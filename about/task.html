Technical Job Task

Aint City Job Application: Technical Task Instructions

As part of the application process for roles at Aint City (Full Stack Developer, Backend Developer, Frontend Developer, Project Manager, DevOps Engineer, Social Media Manager), all applicants are required to complete Part 1 of the following technical task. Developers (excluding Project Manager and Social Media Manager applicants) must also complete Part 2. These tasks demonstrate your ability to engage with Aint Cityâ€™s blockchain-based platform and, for developers, showcase your technical skills.

Part 1: Set Up an Aint Node, Mine AINT, and Mint Anote (All Applicants)

To familiarize yourself with Aint City's blockchain ecosystem, follow these steps to set up a mining Aint Node, mine AINT tokens, and mint Anote using your mined AINT.

  1. Set Up an Aint Node
  2. Mine AINT Tokens
  3. Mint Anote Tokens *

Mint at least 0.1 ANOTE to prove that you're running a node because smaller amounts can be easily earned with Aint Chain mobile mining. It shouldn't take you more than 2 days to mine enough AINT for it.

Submission for Part 1

When applying for a job, send us these:

  • Your Aint address
  • The transaction ID of your Anote minting

Part 2: Develop a Balance Display App (Developers Only)

Note: This task is required for Full Stack Developer, Backend Developer, Frontend Developer, and DevOps Engineer applicants. Project Manager and Social Media Manager applicants are exempt.

Develop a small application that displays the AINT balance of your Aint address from Part 1. You may use any programming language or framework, aligning with your role's expertise (e.g., Golang for backend, React/Vue.js for frontend, or any preferred stack for Full Stack/DevOps).

App Requirements:
  • Create a simple app that queries the Aint Chain to retrieve the AINT balance of your address.
  • Use the Aint Chain API endpoint (nodes.aintchain.com).
  • Display the balance in a user-friendly format (e.g., a CLI output for backend, a web interface for frontend, or either for Full Stack/DevOps).
  • Ensure the app handles basic error cases (e.g., invalid address, network issues).
Implementation Guidelines
  • For Backend Developers: Focus on a server-side solution (e.g., a Golang CLI app or API) that queries the balance and outputs it (e.g., JSON or text).
  • For Frontend Developers: Build a web interface (e.g., React, Vue.js) that displays the balance dynamically.
  • For Full Stack Developers: Combine a backend API with a frontend interface to display the balance.
  • For DevOps Engineers: Create a containerized app (e.g., using Docker) with a focus on deployment and scalability, querying the balance via API.
  • Include basic documentation (e.g., a README) explaining how to run the app.
Submission for Part 2

Host your code in a public GitHub repository and send us these when applying:

  • The GitHub repository URL.
  • A brief description of your app (language/framework used, how it works).
  • Instructions to run the app (e.g., dependencies, commands).
  • A screenshot or short video showing the app displaying your AINT balance.

Notes

  • Submit both parts (if applicable) in a single email to jobs@aintcity.com with your resume and cover letter.
  • If you experience any issues or have any questions about Aint Chain, please show initiative and self-sufficiency by asking for community help in Aint Chain Telegram group.
  • One of reasons for this task is our plan to take over a small blockchain and make it our own by our developers and community of developers running nodes for it which makes Aint Chain perfect for it.
